Live event log

Each lifecycle moment is logged twice on purpose: to show both ways to subscribe. Line one names the subscription: the onX() config callback vs the persona:* DOM event on window. The grey sub-line explains what that moment means.

What you're seeing

  1. persona:script-load fires immediately: the installer ran.
  2. persona:launcher-shown (deferred: true) fires at page load: the real launcher is painted from the ~13 KB critical bundle.
  3. Nothing else fires yet. The heavy conversation panel is not downloaded.
  4. Click the launcher (bottom-right). The full bundle loads, then persona:chat-ready fires and the panel opens.

Each row is timestamped with performance.now() (ms since navigation), so the gap between launcher-shown and chat-ready is the deferral you saved on initial load.
