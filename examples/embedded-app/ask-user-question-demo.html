Auto-advance behavior

When on, picking a single-select pill or pressing Send on free-text jumps to the next question automatically. The final page always waits for an explicit "Submit all" click. Multi-select pages always require Next.

Persistence

Off (default) passes persistState: false — nothing is written to localStorage, so reloading wipes any injected tool calls. On wires up a storageAdapter with a demo-scoped key, so the active question survives a reload. "Reset transcript" still clears either way.

Resume mock — verifying the typing indicator

The widget shows a three-dot typing bubble during the silent gap between the user answering a question and the agent's next streamed token. Pick a preset, answer it, and watch the indicator render below the answer bubble until the simulated /resume response lands.

  • Fast (300ms) — close to a real low-latency backend; the indicator briefly appears then completes.
  • Slow (2s) — easy to inspect; the indicator should remain mounted for ~2 seconds, then a simulated assistant reply appears.
  • Hang — indicator stays mounted indefinitely. Click Reset transcript to recover.
  • Off — falls through to the real (no-op) host; the answer click resolves into a fetch error and the indicator clears via the error path.
