WebMCP — Page tools

This page registers two tools through document.modelContext.registerTool() (via @mcp-b/webmcp-polyfill). Persona snapshots them on every dispatch and ships them as clientTools[]. When the agent calls one, the widget executes it on the page and posts the result back via /resume.

Registered tools

  • search_products — annotated readOnlyHint: true. Filters a static catalog and returns realistic hits (auto-approved by the demo's gate policy).
  • add_to_cart — mutating operation; gated by Persona's native approval bubble, then updates the cart below.

  4. Parallel: "add SHOE-001 and SHOE-007 to my cart" — the model emits two add_to_cart calls in one turn. Each opens its own approval bubble; approve both and the widget batches their results into a single /resume (keyed by per-call id, runtypelabs/core#3878), so both items land in the cart.
