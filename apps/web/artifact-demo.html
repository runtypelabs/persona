Artifact controls

Display mode

Choose where artifact bodies render (features.artifacts.display). Applies to the next streamed artifact.

Streaming body

Tune how inline artifacts stream and settle (features.artifacts.inlineBody). These apply to inline display mode and take effect on the next streamed artifact.

Stream the source code, or show a quiet generating placeholder (inlineBody.streamingView).

Preview completed files in a sandboxed iframe, or always show raw highlighted source, useful when the artifact feeds a code editor (inlineBody.viewMode).

Add a segmented rendered/source toggle to the inline chrome that flips a completed file between its rendered preview and raw source (inlineChrome.showViewToggle), matching the artifact pane's toggle. The control only appears when a rendered preview exists, so it stays hidden for source-only or plain markdown artifacts.

Show a loading state over a slow file preview (filePreview.loading). On keeps the default spinner, Custom swaps in your own indicator via loading.renderIndicator, and Off disables the overlay. It appears only when the preview takes a moment to become ready, so fast artifacts never flash it. Try it with the React file artifact, which pulls React and Babel from a CDN inside the sandbox.

Reserve a fixed-height scroll window so the streaming to complete swap causes no layout shift, or grow with content (inlineBody.height).

Fade the scroll edges of the streaming window (inlineBody.fadeMask).

Keep the streaming window pinned to the newest output (inlineBody.followOutput).

Scroll follows the streaming tail; clip shows the top of the document in a fixed window with no internal scroll, and the body opens the pane when expand is on (inlineBody.overflow).

Cross-fade the streaming to complete swap where supported (inlineBody.transition).

Keep the streamed body inline, or collapse the block to the compact reference card once the artifact completes, the Copilot chiclet / Claude handoff pattern (inlineBody.completeDisplay). The collapse plays as a short height transition; click the collapsed card to open the pane.

Expand toggle

Show an expand/collapse button in the artifact pane toolbar (features.artifacts.layout.showExpandToggle).

Pane appearance

Visual treatment for the artifact column in split mode (features.artifacts.layout.paneAppearance). panel is the bordered sidebar with gap and shadow; seamless is flush with chat (zero gap, no shadow).

Resizable pane

Drag the split handle between chat and the artifact pane in desktop split mode (features.artifacts.layout.resizable). Hidden when the panel is narrow (drawer mode), on viewports ≤640px, or while the pane is expanded.

Custom actions

Add host-defined buttons to the artifact pane toolbar, the in-chat reference card, and the inline block chrome (features.artifacts.toolbarActions, features.artifacts.cardActions, and features.artifacts.inlineActions). The inline "Log" action shows up in the inline chrome title bar.

Inline chrome

File-preview chrome (title bar with copy and expand) around inline blocks in display: "inline" (features.artifacts.inlineChrome). On by default; turn it off for a bare inline body.

Card loading animation

Style the "Generating…" status on the in-chat reference card while an artifact streams. Changes apply live and to the next streamed card.

How it works

Enable artifacts on the widget config:

features: { artifacts: { enabled: true, allowedTypes: ["markdown", "component"], }, },

The buttons here replay the exact artifact_start / artifact_delta / artifact_complete wire frames a real Runtype agent emits, streamed through controller.connectStream(...).

That is what makes the in-chat reference card appear and stream from "Generating…" to done with a Download button, while the side pane fills at the same time. A live agent produces the same frames.

Components must be registered via componentRegistry.register(name, renderer).

Display modes

Where artifact bodies render is controlled by features.artifacts.display:

features: { artifacts: { enabled: true, display: "panel", // panel | card | inline (or { default, byType }) }, },

panel (the default) shows the reference card in the transcript and auto-opens the side pane as the artifact streams. card shows only the card; the pane opens when you click it. inline renders the full artifact preview directly in the transcript, rendered markdown, sandboxed file iframes, or registry components, with no pane involved. The switcher in the rail applies the mode live via handle.update(config); it takes effect on the next streamed artifact.

Streaming body (inline)

When display: "inline", features.artifacts.inlineBody tunes how the body streams and settles:

features: { artifacts: { enabled: true, display: "inline", inlineBody: { streamingView: "source", // source | status viewMode: "rendered", // rendered | source (source: never preview, code editor style) height: 320, // number | "auto" | { streaming, complete } followOutput: true, // tail-follow the streaming source fadeMask: { top: true }, // boolean | { top, bottom } transition: "auto", // auto | none }, inlineChrome: { showViewToggle: true, // rendered/source segmented toggle, only when a preview exists }, filePreview: { loading: true, // true | false | { label, renderIndicator }, appears only when loading takes a moment // loading: { renderIndicator: (ctx) => myEl }, // full custom indicator, null falls back to default }, }, },

A numeric height reserves a fixed, internally scrolled window for the streaming source and pins the completed iframe to the same height, so the streaming to complete swap causes no layout shift. "auto" grows with content instead. streamingView: "status" shows a quiet generating placeholder in place of the streaming code, and transition: "auto" cross-fades the completion swap where the View Transitions API is supported. These only affect display: "inline". The controls in the rail apply them live via handle.update(config), landing on the next streamed artifact.

Card loading animation

The reference card's "Generating…" status animates while the artifact streams. Configure it on features.artifacts:

features: { artifacts: { enabled: true, loadingAnimation: "shimmer", // none | pulse | shimmer | shimmer-color | rainbow loadingAnimationDuration: 2000, // ms per cycle loadingAnimationColor: "#0ea5e9", // shimmer-color only loadingAnimationSecondaryColor: "#3b82f6", // shimmer-color only }, },

The default is shimmer. Character-by-character modes (shimmer, shimmer-color, rainbow) stagger a per-character animation; pulse fades the whole status; none disables it. All modes honor prefers-reduced-motion. The controls in the rail update these values live via handle.update(config), so the running widget re-renders without losing the transcript.
