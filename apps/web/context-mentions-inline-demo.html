Inline Context Mentions

The same @ mentions, rendered inline: contextMentions.display: "inline" swaps the composer for a contenteditable surface, and selecting a mention drops an atomic token into the sentence (Slack/Linear/Cursor style) instead of the default chip row. Same sources, same resolve, same wire.

Try it

  • Type @ and pick a file — a styled token lands in your text
  • Keep typing around it; the token stays whole and moves with the caret
  • Backspace next to a token deletes the whole token, not a character
  • Send — the bubble shows the @token inline, and the resolved file body still reaches the model (echoed below the reply)

Notes

How it works

Set contextMentions.display: "inline" (default is "chip") alongside the same sources you'd use in chip mode. Inline mode reuses the entire chip pipeline — the source search/resolve, the menu, abort-on-remove, and the submit bundle — and only changes the display surface. The contenteditable engine ships as a separate ~3 kB lazy chunk that loads only when display: "inline" is set, so chip-only and feature-off installs pay nothing.

Token styling

Use the Token styling toggle above the preview to compare the three levels of control over how a token looks:

  • Colored — give each item a color and the whole pill (tint + icon + text) recolors. Here it varies by file type, the pluggable "color per type" knob.
  • Monochrome — omit color and tokens fall back to the theme accent, one uniform look.
  • Custom — a renderMentionToken hook replaces the token DOM entirely (this variant renders a bordered, monospace, dot-prefixed chip). Hosts can also theme by source in CSS via .persona-mention-token[data-mention-source="files"].

Config snippet

import { createStaticMentionSource } from "@runtypelabs/persona"; const config = { contextMentions: { enabled: true, display: "inline", // atomic tokens in the sentence sources: [ createStaticMentionSource({ id: "files", label: "Files", items: [ { id: "app", label: "App.tsx", iconName: "file-code", color: "#2563eb" }, ], resolve: (item) => ({ llmAppend: readFile(item.id) }), }), ], }, };

What reaches the model

Inline mode is context attachment, not entity substitution. The one model-visible difference from chip mode is the prose: chip mode strips the @query on select, while inline mode keeps the readable @label in the sentence (matching Cursor/ChatGPT/Claude attachments, so where the file is referenced is preserved). The resolved content that reaches the model is the same in both modes: each mention's resolved body is merged into the LLM content via applyMentionBundle's [label]\n<body> concatenation, exactly as chip mode.
