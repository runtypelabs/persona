Bring-Your-Own Voice

Plug a custom speech engine into Persona via voiceRecognition.provider.custom. This demo wraps the browser's Web Speech API as a VoiceProvider and dictates messages through the composer mic.

How the slot works

voiceRecognition: { enabled: true, provider: { type: 'custom', // a VoiceProvider instance, or a () => VoiceProvider factory custom: () => createWebSpeechVoiceProvider(), }, }

The adapter emits each final transcript via onResult; Persona sends it as a user message and the assistant reply streams back over the normal chat path. See src/custom-voice-provider.ts.

Replies are spoken back with textToSpeech: { enabled: true, provider: 'browser' }: a separate subsystem that pairs with any input provider, so an STT-only custom provider still gets voice output for free.

Notes

Implementing a provider

A VoiceProvider needs connect/disconnect, startListening/stopListening, and the onResult/onError/onStatusChange registration methods. Report 'listening'/'processing'/'idle' so the mic button reflects state.

STT-only providers (like this Web Speech adapter) deliver a final transcript via onResult. A full-duplex provider that also speaks replies can additionally drive onTranscript and onMetrics, the way the realtime runtype provider does.

Speech in vs. speech out

Voice input (this custom provider) and voice output (textToSpeech) are independent. This demo pairs the custom STT provider with browser TTS so the reply is read aloud: the same textToSpeech block works with the browser or runtype input providers too.

A full-duplex provider (like runtype) instead streams its own spoken audio and skips browser TTS; in that case the provider owns playback via a VoicePlaybackEngine.

Requirements

Needs the demo proxy running (pnpm dev) for the assistant reply, and a Chromium-based browser for Web Speech. The mic button renders even without Web Speech because the provider type is custom.
